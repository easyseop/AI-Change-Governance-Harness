## 판정 요약

**보정요청 1건(R-1) — 코드 브랜치 머지 보류.** 제출된 검증 주장은 **전부 정확히 재현**된다(진입점 13/13 · selftest 96/96 + mutation 161 · `bash -n` PASS · 임시파일 누수 없음). 크래시도 실제로 사라진다. AC#3(콘솔 `missing_expected`)·AC#5(`out_of_scope` 부재 단언)는 **그대로 수용**한다.

문제는 **크래시를 없앤 방법**이다. `run.sh` 가 intent 부재 시 `allowed_paths: ["**"]` 짜리 **가짜 change-intent 를 합성해 게이트에 주입**한다. 그 결과 ① 감사카드가 **존재하지 않는 선언을 검사해 통과했다고 위조**하고 ② **`change-intent.yaml` 을 지우면 의도층 전체(forbidden_paths·expected_paths)가 통과로 무력화**된다. 이건 과탐이 아니라 **under-detection** 이라 §2B 필수질문 = **예** → 비차단 불가.

**그리고 이 결함의 1차 책임은 내(Claude) AC 에 있다.** AC#2 가 "intent 없는 repo → exit 0" 을 요구했는데, 그 전제("exit 1 = 크래시 산물")가 **틀렸다**. 실제로 bash≥4.4 에서는 크래시 없이 게이트가 돌고 **의도적으로 `blocked`(exit 1) 판정**을 낸다(`load_intent` → `FileNotFoundError` → 카드 `reasons: [의도 선언 누락…]`). Codex 는 틀린 AC 를 충실히 구현했다. **AC#2 를 아래와 같이 정정**하고 보정을 요청한다.

## R-1: 합성 intent 주입 (보정요청)

### 결함 1 — 감사카드 위조 (정직성 위반)

`--change-intent <합성파일>` 을 받은 게이트는 그것이 합성인지 알 수 없다. 산출 카드(실측):

| 필드 | 값 | 문제 |
|---|---|---|
| `verdict` | `pass` | 선언이 없는데 "의도이탈 없음" |
| `intent_check.status` | `pass` | 검사한 적 없는 층이 통과로 기록 |
| `changed_files[].in_allowed_paths` | `true` | **선언이 없는데 "허용경로 안"으로 위조** |
| `coverage_statement.checked` | `check-change-intent: "changed paths against **declared** allowed_paths, forbidden_paths, and optional expected_paths"` | **선언 대조를 수행했다고 명시 주장** |
| `reasons` | `[]` | 침묵 |

카드 어디에도 선언이 러너에 의해 합성됐다는 흔적이 없다(`policy_sha` 는 zones·routing 만 담아 임시파일 해시조차 안 남는다). 콘솔에는 "생략" 안내가 뜨지만 **콘솔은 휘발성이고 카드가 감사 산출물**이다. CI 에 남는 건 카드다.

대조군 — 합성 없이 게이트가 직접 판단하면 카드는 **정직**하다: `verdict: blocked` · `reasons: ['의도 선언 누락: change-intent.yaml 파일을 찾을 수 없습니다.']` · `coverage_statement.checked: []`(=아무것도 검사 안 함). 이 브랜치는 **정직한 `checked: []` 를 "3개 게이트 검사 완료"라는 거짓 주장으로 바꾼다.** "지적 없음"을 "분석·안전"으로 오해하게 두지 않는다는 원칙(TASK-019 계보·조용한 통과 금지)의 정면 위반.

### 결함 2 — 의도층 우회 + bash≥4.4 탐지 회귀

**fresh 적대 repo**(픽스처 밖·`allowed_paths: ["app/**"]` · `forbidden_paths: ["app/keys.py"]` · `expected_paths: ["app/must_patch.py"]`)에서 **forbidden 파일을 수정하고 expected 파일은 안 건드린** diff:

| 실행 | intent 존재 | **intent 파일 삭제 후** |
|---|---|---|
| main (bash 3.2) | exit 1 BLOCKED | **크래시** exit 1 · 카드 없음 (= O-A 결함) |
| main + 안전관용구만 (**= bash≥4.4 에서 main 의 현재 실동작**) | exit 1 BLOCKED | **exit 1 BLOCKED** · 카드 생성 · 정직 |
| **이 브랜치** | exit 1 BLOCKED | **exit 0 PASS** · 카드 `verdict: pass` |

즉 **`change-intent.yaml` 을 삭제하는 것만으로** forbidden_paths 차단과 expected_paths 부재탐지(TASK-027/028 패치 생존성 층 전체)가 **통과로 사라진다**. "선언 안 하면 무제한 허용" = 의도층 우회 경로. 리눅스 CI(bash≥4.4)에서는 크래시가 아니라 **BLOCKED → PASS 판정 완화 회귀**다. TASK-033 은 크래시 수정 태스크이지 판정 완화 태스크가 아니다.

### 결함 3 — AC#1 안전관용구가 dead code (가드 미실증)

합성 주입 때문에 `INTENT_ARGS` 는 **절대 비지 않는다** → `${INTENT_ARGS[@]+"${INTENT_ARGS[@]}"}` 는 한 번도 빈 배열로 확장되지 않는다.

**rig 실증**: 안전관용구를 `"${INTENT_ARGS[@]}"` 로 되돌리고 합성만 남기면 → **진입점 13/13 PASS 유지**. AC#1 이 요구한 바로 그 메커니즘이 **죽어도 아무 테스트도 안 운다.** 반대로 합성만 제거하고 관용구를 남기면 → 크래시 없이 정상 동작하되 신규 케이스만 FAIL(12/13) = **실제로 크래시를 막는 건 관용구, 테스트가 지키는 건 합성**. 가드와 테스트가 어긋나 있다.

## AC#2 정정 (내 오류 · Claude)

**정정 전(틀림)**: intent 없는 repo → **exit 0** + 생략 문구 + 카드 생성.
**정정 후**: intent 없는 repo → **크래시 없이 게이트 자신의 판정이 그대로 흐른다**(현행 계약 = `blocked`/exit 1·정직한 `reasons: [의도 선언 누락…]`·`coverage.checked: []`) + **카드 생성** + **2층/3층/메타층 전부 실행**.

크래시 결함의 실체는 "exit 1" 이 아니라 **카드 미생성·후속 3개 층 미실행·판정 없는 죽음**이었다. 그 셋만 고치면 된다. **판정 자체는 건드리지 않는다.**

## 보정안 (실증 완료 — RIG2 로 확인)

1. **합성 intent 주입 전량 제거**(`TEMP_INTENT`·`mktemp`·`printf`·`rm -f`). `run.sh` 의 델타는 **AC#1 안전관용구 1줄 + AC#3 콘솔 grep 1줄**로 축소.
   ```bash
   INTENT_ARGS=(); [ -n "$INTENT" ] && INTENT_ARGS=(--change-intent "$INTENT")
   run_gate "generate-change-evidence" "0 1 2" "$G/generate-change-evidence.py" "$RANGE" \
     --sensitive-zones "$ZONES" --approval-routing "$ROUTING" \
     ${INTENT_ARGS[@]+"${INTENT_ARGS[@]}"} --repo .
   ```
2. **진입점 케이스 재작성** (`no-intent-*`) — 단언을 **크래시 결함 자체**로 정조준:
   - `/bin/bash`(3.2) 실행 · 출력에 **`unbound variable` 부재**
   - **카드 파일 생성 + 비어있지 않음**
   - **`[2층]`·`[3층]`·`[메타]` 층 전부 실행됨**(현행 단언 유지 — 이 부분은 좋다)
   - `rc = 1` (현행 계약 고정. TASK-034 가 2 로 정규화하면 그때 함께 갱신)
   - **카드 정직성 단언**: `reasons` 에 `의도 선언 누락` 존재 **AND** `verdict: pass` **아님** **AND** `in_allowed_paths: true` **부재** ← 합성 주입이 재유입되면 자동 검출(위조 방지 가드)
3. **음성검증 2종 필수 기록**: ① 안전관용구를 `"${INTENT_ARGS[@]}"` 로 되돌림 → **신규 케이스 단독 FAIL**(관용구가 load-bearing 임을 증명 — 현재는 13/13 유지라 실패) ② 기대 rc 변조 → 단독 FAIL.
4. **콘솔 문구 정정**(AC#2 부수): 151행 `(change-intent.yaml 없음 — 의도이탈 층은 생략)` 은 **실동작과 모순**이다(층은 생략되지 않고 게이트가 차단 판정을 낸다 — 위 SAFE 실측). "의도 선언 누락 — 카드 게이트가 미선언으로 판정" 류의 사실 문구로. 판정 로직 무변경·표시만.
5. **AC#3·AC#5 는 현행 그대로 유지**(검증 통과 — 재작업 금지).

**범위 밖(건드리지 말 것)**: `kit/gates/*`·dev 게이트·정책. "미선언 → blocked vs approval_required" 판정 정규화는 **TASK-034**(별도·Claude 정책 결정 완료)로 분리했다.
