## 🟠 R-2 (블로킹·잔여) — bare 호출이 method 스코프 안에서 동명 클래스메서드로 오해소

**위치**: `.harness/gates/extract-callgraph.py` `CallVisitor.visible_local_names()`(231–235). 클래스 확장 루프는 제거됐으나 `for index … ".".join(self.parents[:index] + [name])` 의 **parents-prefix 확장이 여전히 클래스명을 포함**한다 — `CallVisitor.visit_ClassDef`(179)가 클래스명을 `self.parents` 에 push하기 때문.

**기전**: caller 가 **메서드**면 `parents` 에 클래스명이 있어 bare 호출이 `{Class}.{name}` 후보를 만든다. `resolve_repo_function` 의 `sorted(set(candidates))[0]` 이 대문자 클래스명(0x43)을 소문자 모듈함수보다 먼저 pick → **형제 메서드로 오해소**. Python 의미상 **메서드 본문의 bare 이름은 클래스 스코프를 건너뛰고**(LEGB — 클래스 스코프 제외) 모듈/전역으로 바인딩되므로 이 해소는 **항상 틀림**.

**fresh 실증(픽스처 밖)** — `app/m.py`:
```python
def sink():                       # 모듈 함수 (실제 sink 가정)
    return 1
class C:
    def sink(self):               # 동명 형제 메서드
        return 2
    def caller(self):
        return sink()             # ← bare: Python -> 모듈 sink
```
- 산출 엣지: `app.m.C.caller -> app.m.C.sink` (**틀림 · 조용한 오염**). 정답 `app.m.C.caller -> app.m.sink` **부재**, `unresolved_calls` 에도 **없음**.
- `self.sink()` 는 정직히 unresolved(정합) — bare 호출만 오염된다.
- R-1(누락) 보다 나쁨: 누락이 아니라 **틀린 엣지 주입**.

**왜 R-1 과 동일한 거버넌스 하류 구멍(§2B 필수질문 = 예)**: 모듈함수 `sink` 가 실제 sink 면, 이를 bare 로 호출하는 **메서드 caller 가 TASK-024 역도달 상류에서 누락**(정답 엣지 유실)되고 동시에 형제 메서드로 **거짓 엣지가 주입**된다 → 민감 변경 미포착. A-0018 R-1 이 없애려던 실패모드("진짜 caller→sink 엣지 유실")를 method-caller 경로에서 그대로 재현. 비차단 불가.

**거짓 커버리지 함정**: 신설 `overloads.py` 픽스처는 **모듈-caller `bar` 만** 단언(caller 가 모듈함수). method-caller 변형은 미커버라, "동명 오버로드 상설 픽스처를 추가했다"는 표기가 §2B 가 명령하는 커버리지를 **절반만** 채우면서 전체를 채운 듯한 확신을 준다.

**보정(택1 · ①권장 · 소델타)**:
1. `visible_local_names` 에서 **클래스 스코프를 건너뛴 parents-prefix만** 후보화한다(bare 이름은 클래스 스코프에 미바인딩). 예: prefix 가 `self.class_names` 에 있으면 `continue`. **Claude 사본 실증**: 이 3줄 변경으로 `tests/run-tests.sh` **81/81 유지** + method-caller 케이스가 `C.caller -> 모듈 sink` 로 교정됨.
2. method-caller 동명 충돌 시 pick-smallest 대신 **모듈/전역 우선**(또는 union). (①보다 큼.)
3. **상설 회귀 픽스처를 method-caller 변형으로 확장**: 같은 모듈에 모듈 `sink` + `class C{def sink; def caller: return sink()}` → 엣지가 **모듈 sink 로** 가고 `C.caller -> C.sink` 는 **없음**을 단언. (§2B "동명 오버로드" 상설화 완성.)
