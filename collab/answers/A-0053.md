- **AC#13 의 대안**: `inferred_edges` 대신 **엣지에 출처 태그**(`source: observed|inferred`)를 달아 그래프 레벨에서 관리하는 편이 더 정확하다 — 지금은 같은 `(caller, callee)` 가 두 출처를 가질 때 관측이 무조건 이긴다. 다만 **그 병합 규칙이 안전 방향**(관측 우선 = 카드가 덜 겁주지 않음)이고 **7줄 vs 추출기 스키마 변경**이라 **현행이 옳은 균형**이다. 대규모 리팩터를 강요하지 않는다.
- **하류(TASK-038 킷)**: 이제 킷이 스냅샷할 대상은 **탐지 + 회귀보호 + 정직한 표식** 3종이 다 갖춰진 상태다. `inferred` 키는 **카드 렌더에 노출돼야** 의미가 있으므로 TASK-038 이 킷 README/`manifest.yaml` 에 **`inferred` 표식의 뜻**("보수 추정 홉 포함 — 실제 호출 관측 아님")을 적어야 한다. 안 적으면 배포 킷 사용자에겐 그냥 노이즈 키다. → TASK-038 AC 에 추가.
- **하류(감사카드)**: `dead_ends=` 가 다시 원인 함수를 이름으로 준다 ⇒ 사람 리뷰어가 **"왜 승인요구인가"** 를 30초 안에 읽는다(CLAUDE.md §2C). O-22 가 남아 있어 **무관한 이름이 섞일 수 있음**은 킷 README 에 한 줄로 적어두는 편이 안전하다.
