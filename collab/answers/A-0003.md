## Q1. 문법 — AST 리터럴로만, 2가지 형태 (실행·추정 금지)

### (a) 함수/클래스/메서드/중첩 함수: **데코레이터 형태 (유일한 부착 문법)**
```python
@gov(level="protected", reason="이체 한도 검증 — 자금이체 경계", owner="core-banking")
def validate_transfer_limit(...): ...

@gov(level="frozen", reason="정산 원장 기록")
class LedgerWriter:
    @gov(level="frozen", reason="원장 커밋")
    def commit(self): ...
```
- 인식 규칙: 데코레이터 **정규화 이름이 `gov` 또는 `*.gov`** (TASK-005 데코레이터 resolve 규약 재사용 — `Name`/`Attribute`/`Call` 대응).
- **keyword 인자만** 허용, 값은 **문자열 리터럴만** (f-string·변수·호출·연결식 불가 → Q5 `unresolved`).
- 근거: ① AST 만으로 결정적 파싱(주석/독스트링 마커는 tokenize 필요 + 부착 규칙 모호 → MVP-1 **비지원, 명시 거부**) ② TASK-005 가 이미 데코레이터 목록을 추출하고 ③ TASK-006 AC5 가 **데코레이터 줄 변경 → 해당 함수 매핑**을 이미 보장 — `@gov` 자체를 고치는 변경도 그 함수의 변경으로 잡힌다(기존 인프라 재사용).
- 런타임: 실제 저장소는 no-op `def gov(**kw): return lambda f: f` 를 vendoring 하면 됨. **게이트는 import 여부와 무관하게 AST 로만 파싱**(fixtures 에 gov 정의 불요).

### (b) 모듈 레벨 코드: **`__gov__` 톱레벨 리터럴 dict 할당**
```python
__gov__ = {"level": "protected", "reason": "정산 배치 설정 상수"}
```
- 모듈 레벨(`<module>` touched, TASK-006)은 데코레이터 부착이 불가능하므로 이 형태로.
- 톱레벨 직속 `Assign`, 대상 단일 `__gov__`, 값은 **문자열 key/value 리터럴 dict** 만.

## Q2. 필드
| 필드 | 필수 | 타입/제약 |
|---|---|---|
| `level` | ✅ | `"frozen" \| "protected" \| "watched"` |
| `reason` | ✅ | 비어있지 않은 문자열 (사람 리뷰어가 30초에 이해할 문장) |
| `owner` | 선택 | 문자열 (팀/담당) |
- 그 외 필드는 전부 **unknown_field 검증오류**(기록 후 무시). `capabilities`·`data` 등은 **Phase C(TASK-010) 로 명시 이연** — 지금 정의하지 않는다(임의 의미 부여 금지). unknown_field 오류가 조기 사용을 잡아준다.

## Q3. 값·심각도·exit code
- 어휘는 `sensitive-zones.yaml` 과 **동일**: `frozen > protected > watched` (TASK-009 "policy 재사용").
- **TASK-008 파서 게이트는 보고 전용** — Phase A 게이트와 동일하게 **항상 exit 0**, 판정 없음(사용법 오류 제외).
- 판정은 **TASK-009**: 변경 함수(TASK-006 매핑 ∪ TASK-007 분류)의 effective level 이
  `frozen → blocked(exit 1)` / `protected → approval_required(exit 2)` / `watched → 경고+pass(0)` / 무주석 → 다른 게이트 결과대로.

## Q4. 부착(scope) 규칙 — **엄격 승계(stricter-wins), 하향 우회 불가**
1. 데코레이터는 **자기가 감싼 def/class 하나**에 부착 (AST `decorator_list`).
2. **effective_level = max(자기 주석, 둘러싼 클래스/함수 주석들, 모듈 `__gov__`)** — frozen > protected > watched > 없음.
   내부 주석은 레벨을 **올릴 수만 있고 내릴 수 없다** (frozen 클래스 안 메서드에 `watched` 붙여 우회 → 불가, effective 는 frozen 유지 + 관찰 기록).
3. 동명 함수(getter/`@x.setter`·`@overload`·조건부 재정의): **각 def 가 자기 주석을 가짐.** 식별 키는 TASK-007 AC5 와 동일 — `(정규화이름, 데코레이터셋/라인순서)`. **이름 단독 매칭 금지.**
4. 모듈 레벨 변경(`<module>`)은 `__gov__` 로 판정.

## Q5. 검증 실패 처리 — 파일 단위 격리 + 보수 기본값 (silent pass 금지)
TASK-008 출력에 per-annotation `errors[]` 로 기록. **TASK-009 는 변경 함수에 invalid/unresolved 주석이 있으면 최소 `approval_required`** (조용히 통과 금지).
| 케이스 | 오류 코드 | 파서 처리 |
|---|---|---|
| `reason` 누락/빈 문자열 | `missing_reason` | level 유효하면 주석은 **유효 취급**(level 유지)+오류 기록 |
| `level` 미지 값 | `invalid_level` | effective 산정 시 **protected 로 보수 취급** |
| `level` 누락 | `missing_level` | protected 보수 취급 |
| 비-리터럴 값(변수·f-string·호출) | `unresolved` | 값 추정 금지, protected 보수 취급 |
| unknown 필드 | `unknown_field` | 해당 필드 무시, 주석은 유효 |
| 한 def 에 `@gov` 중복 | `duplicate` | **가장 엄한 유효 level** 채택 |
| positional 인자 | `invalid_syntax` | unresolved 와 동일 보수 취급 |
| `__gov__` 재할당 다중/비-dict/비-리터럴 | `invalid_module_gov` | 유효 할당 중 가장 엄한 것, 전부 무효면 protected 보수 |
| 파일 문법오류 / 비-UTF8 | `parse_error` / `unreadable` | **파일 단위 격리, 형제 파일 보존, exit 0** (TASK-013 계보) — TASK-012 fail-closed 가 받음 |

## Q6. 출력 스키마 — **별도 게이트** `.harness/gates/extract-gov-annotations.py`
TASK-005 인벤토리 **확장 금지** — 그 출력은 이미 TASK-006/007 의 하류 계약으로 고정(스키마 변경 = blast radius). 같은 정규화 이름·동명 키 규약을 재사용해 join 가능하게 한다.
- 입력: 단일 `.py` 파일 경로 또는 stdin (TASK-005 와 동일 모드). base/head 양측 적용은 TASK-009 호출측 책임(`git show` — TASK-007 패턴).
- 결정적 + `--json`, exit 0.
```json
{
  "path": "app/core.py",
  "module": {"level": "protected", "reason": "...", "line": 3, "errors": []},
  "annotations": [
    {"name": "LedgerWriter.commit", "def_line": 42, "gov_line": 41,
     "order_key": 0, "decorators": ["gov", "staticmethod"],
     "level": "frozen", "reason": "원장 커밋", "owner": null,
     "effective_level": "frozen", "errors": [], "unresolved": false}
  ],
  "parse_error": false, "unreadable": false
}
```
(`module` 없으면 `null`. `order_key` = 같은 정규화 이름 내 등장 순서 — TASK-007 매칭 키와 동일 의미.)

## Q7. 테스트 수용기준 (리뷰 체크리스트로 사용)
1. 기본 부착: 일반 함수·`async def`·메서드·중첩 함수 각각 `@gov` 정상 파싱 + `__gov__` 모듈 케이스.
2. **🔴 엄격 승계**: frozen 클래스 안 `watched` 메서드 → `effective_level: frozen` (하향 우회 차단 실증).
3. **🔴 고정 적대 세트**(CLAUDE.md §2B — 상설 회귀 픽스처): ① property getter/setter 중 **setter 에만** `@gov` → order_key 로 정확 부착(이름 단독 매칭이면 오판) ② `@overload` 동명 ③ 조건부/중첩 def.
4. 검증 실패 각 1 케이스 이상: `invalid_level`·`unresolved`(변수 값)·`duplicate`·`missing_reason`·`unknown_field`·positional.
5. fail-safe: 문법오류 파일 → `parse_error` 격리·exit 0 / 비-UTF8 파일 → `unreadable` 격리 (+ 다중 파일 러너 케이스면 형제 보존 — TASK-013 픽스처 패턴 재사용).
6. 결정성: 동일 입력 2회 md5 동일.
7. 음성검증 가능성: 기대값 변조 → 러너 FAIL (리뷰 시 재현한다).
