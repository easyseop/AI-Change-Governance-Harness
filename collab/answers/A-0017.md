- **성격**: 신규 태스크 아님 — **비차단 소규모 동반작업**(TASK-023 착수 세션에서 함께 처리 권장, ~10분).
- **왜 Codex 몫인가**: `kit/run.sh` 는 D-050 에서 Codex 저자로 확정된 파일. Claude 가 안내 echo 를 직접 넣어 자기머지하려다 거버넌스 분류기에 차단됨(상호견제 정상 작동, 2026-07-15). 스펙만 넘긴다.
- **배경(E2E 실측·D-051)**: 외부 오픈소스 4종 테스트 중 도입자 혼동 지점 2건 발견 — 킷 버그 아님, 안내 부재.

## 구현 요구 (판정 무영향 — echo 만, verdict·exit code 경로 0-diff)

### ① change-intent 스키마 오기 안내
- **증상**: `allowed_paths` 를 top-level 에 두면(정식은 `change_intent:` 아래 중첩) 게이트가 빈 선언으로 읽어 **전 변경 out_of_scope(승인요구)**. fail-safe 라 구멍은 아니나 "왜 다 승인요구?" 혼란(검증자 본인도 E2E 중 두 번 헛짚음).
- **요구**: `run.sh` 가 대상 repo 의 `change-intent.yaml` 존재 시 yaml 을 판독해 —
  - top-level 에 `allowed_paths`/`forbidden_paths`/`requirement_id` 가 있는데 `change_intent:` dict 가 없거나 그 안 `allowed_paths` 가 비면 → `⚠ change-intent.yaml: allowed_paths 가 top-level 에 있음 — 'change_intent:' 아래 중첩이 정식 스키마(현재 빈 선언으로 읽혀 전 변경이 승인요구됨). 예: policies/change-intent.example.yaml`
  - `change_intent:` 는 있는데 `allowed_paths` 가 비면 → `⚠ ... allowed_paths 가 비어있음 — 전 변경이 out_of_scope(승인요구)로 처리됨`
  - yaml 파싱 실패 시 조용히 스킵(게이트가 자체 처리 — 이 안내가 게이트 판정을 선점하면 안 됨).
- **판정 불변 필수**: 오기 형태여도 최종 verdict 는 지금과 동일(승인요구 fail-safe 유지). 안내만 추가.

### ② 감사카드 오염 안내
- **증상**: 기본 `--output` 이 대상 repo 안 상대경로(`change-evidence.yaml`) → `git add -A` 시 커밋에 섞여 다음 diff 오염(실측 재현).
- **요구**: `OUT` 이 상대경로면 → `⚠ 감사카드가 대상 repo 안에 생성됨($OUT) — 커밋에 섞이면 다음 diff 오염. --output <외부경로> 또는 .gitignore 등록 권장`. 기본 동작(경로·파일 내용)은 불변.

## 수용 기준
1. 경고 3케이스 실증: 오기 스키마→⚠① / 정식 스키마→무경고·판정 불변 / 기본출력→⚠②.
2. `kit/selftest.sh --quick` 무회귀(러너 스위트 + 진입점 적대 전부 PASS).
3. verdict·exit code 경로 0-diff (기존 진입점 적대테스트가 가드).
4. `bash -n` OK.
