## R-4 (차단 · **신규 유입** · 배포 최전선 판정 회귀) — 킷 기본 사용법에서 **모든 변경이 오차단**된다

### 결함

`generate-change-evidence.py` 의 argparse:
```python
"--language-routing", default="policies/language-routing.yaml",   # ← 상대경로 기본값
```
그런데 **`kit/run.sh` 는 `--language-routing` 을 아예 넘기지 않는다**(`grep -c language kit/run.sh` = **0**). run.sh 는 `cd "$REPO"` 후 게이트를 돌리므로, 이 상대경로는 **킷의 `$POL` 이 아니라 대상 repo 의 cwd 기준**으로 해석된다. 형제 정책 4종(`ZONES`·`CAPS`·`ROUTING`·`SINKS`)은 전부 `$POL/...` 절대경로로 명시 전달되는데 **language-routing 만 누락**됐다.

여기에 R-2 의 fail-closed 가 얹히면서: argparse **기본값은 항상 "지정됨"** 이므로 `not language_routing`(무음) 분기에 절대 걸리지 않고, 대상 repo 에 `policies/language-routing.yaml` 이 없으면 **무조건 `errors` → `blocked`** 다. 내 보정안의 "**명시**됐는데 부재" 와 "기본값" 의 구분이 구현에서 붕괴한 것이 핵심 원인이다.

### 실증 (fresh repo · 픽스처 밖 · 무해한 Python-only 변경 · intent 정상 선언)

| 실행 | exit |
|---|---|
| `main`(현행 기준선) | **0 PASS** |
| 보정 전(`65e1164`) | **0 PASS** |
| **보정 후(`48ed2be`)** | **1 🔴 BLOCKED** ❌ |

카드 실측: `verdict: blocked` · `reasons: - 'language routing policy missing: policies/language-routing.yaml'`.
**README 의 1번 사용법**(`./run.sh <base>..<head> --repo <대상repo>` — 킷 동봉 정책 사용)이 정확히 이 경로다.

**`--policies` 로도 구제되지 않는다**: `--policies <kit>/policies` 를 명시해도 language-routing 은 그 값을 **전혀 참조하지 않아** 여전히 exit 1(실측). 또한 run.sh 의 필수정책 preflight 루프에도 `language-routing.yaml` 이 **빠져 있어** 사전 진단조차 안 된다.
유일하게 통과하는 조건은 대상 repo **안에** `policies/language-routing.yaml` 이 존재할 때다(실측 exit 0).

### 왜 전 스위트가 초록인데 안 잡혔나 (미측정 ≠ 무발견)

`kit/tests/run-entrypoint-tests.sh` 의 fixture 가 `mkdir -p "$repo/policies"` 후 `cp "$KIT/policies/"*.yaml "$repo/policies/"` 로 **킷 정책 전량을 대상 repo 안에 복사**한다. 이번 sync 로 `language-routing.yaml` 이 킷 정책에 들어오면서 **테스트 repo 만 우연히 상대경로가 해소**된다. 즉 13/13 은 결함을 **측정하지 않았을 뿐**이다. dev 쪽도 repo 루트에 `policies/language-routing.yaml` 이 있고 `.harness/run.sh` 자체가 없어(게이트 직접 호출) **dev 에서는 영원히 재현되지 않는다** — 전형적 dev/배포 divergence.

### 왜 비차단으로 못 미루나 (§2B 필수질문 = **예**)

거버넌스 목적에 직접 구멍을 낸다. under-detection 은 아니지만(차단 방향), **모든 변경이 상시 🔴 로 뜨면 BLOCKED 신호가 정보량 0 이 되어** 승인자가 카드를 무시하거나 킷을 꺼버린다 — 이 repo 가 지켜온 "판정 회귀 없음"·"배포 최전선 정직성" 불변식을 정면으로 깬다. 게다가 **`main` 대비 명백한 판정 회귀**(0 → 1)다.

### 보정안 (blast radius 작음 — run.sh 2줄 + 픽스처 1건)

1. `kit/run.sh` 의 `generate-change-evidence` 호출에 **`--language-routing "$POL/language-routing.yaml"` 명시 추가** — 형제 정책 4종과 동일 규약(따라서 `--policies` 오버라이드도 자동으로 존중됨).
2. run.sh 의 **필수 정책 preflight 루프에 `$POL/language-routing.yaml` 추가** — 정책 디렉토리에 없으면 카드 blocked 가 아니라 기존 관례대로 `필수 정책 파일 없음` **exit 2** 로 명확히 진단.
3. **argparse 기본값을 `None` 으로 내리는 것을 권장** — "미지정" 과 "지정됐는데 부재" 의 구분이 R-2 설계의 전제였고, 상대경로 기본값이 그 전제를 무너뜨린다. (1·2 를 하면 실사용 경로는 항상 명시되므로 기본값은 안전망일 뿐이다.)
4. **진입점 회귀 케이스 신설(필수)**: 대상 repo 에 `policies/` 를 **두지 않고** 킷 동봉 정책으로 돌리는 케이스 — 무해 변경이 **exit 0** 이고 카드에 `language routing policy missing` 이 **없을 것**을 단언. 현재 fixture 는 정책을 repo 안에 복사하므로 **이 경로를 덮는 케이스가 0건**이다.
5. **음성검증**: 1번 배선을 되돌리면 4번 케이스가 단독 FAIL 하는지 기록(가드 load-bearing 증명).
