## 🔴 R-1 (보정 사유): 동일 주석 *안에서* `level` 키 중복 → last-wins 다운그레이드

A-0003 Q5 의 `duplicate` 행("가장 엄한 유효 level 채택")은 **@gov 데코레이터 2개** 케이스만 구현됐고(그쪽은 정확 — `duplicate_gov` 픽스처 실증), **하나의 주석 안에서 같은 필드가 2번 나오는** 두 경로가 last-wins 로 뚫린다:

**(a) 데코레이터 중복 keyword** — `@gov(level="frozen", level="watched", reason="...")`
- 전제 확인: **`ast.parse` 는 중복 keyword 를 거부하지 않는다**(SyntaxError 는 `compile`/import 단계 — 호스트 3.11.4 실측). 게이트는 AST 만 보므로 이 입력이 살아서 들어온다.
- 게이트 실측: `level=watched, errors=["duplicate"], unresolved=false` — `parse_gov_call` 의 필드 대입(`if field == "level":`)이 무조건 덮어써 **마지막 값(watched) 채택**. `seen_fields` 는 오류만 기록하고 값 채택엔 관여 안 함.
- 영향: frozen 이어야 할 판정이 watched 로 강등. TASK-009 가 `duplicate` 를 invalid 로 간주해도 최대 approval_required — **frozen=blocked 가 무력화**. (이 형태는 import 시엔 SyntaxError 라 실행 코드론 자멸적이지만, 게이트 판정은 소스 기준이고 실행 여부에 의존하면 안 된다 — 같은 지점 수정이라 함께 고친다.)

**(b) `__gov__` dict 리터럴 중복 키** — `__gov__ = {"level": "frozen", "level": "watched", "reason": "..."}`
- 이쪽은 **합법·실행 가능한 Python** 이다(dict 리터럴 중복 키 허용, 런타임도 last-wins).
- 게이트 실측: `level=watched, errors=[]` — **오류 기록 0건의 silent downgrade.** `parse_module_gov_value` 의 `zip(keys, values)` 순회가 마지막 값으로 덮어쓴다.
- 영향: Q5 "silent pass 금지" 정면 위반. 무기록이라 TASK-009 의 "invalid/unresolved → 최소 approval_required" 안전망조차 못 받는다. TASK-009 base∪head max 가드도 신규 파일·base 부터 오염된 파일에는 무력 — **이 게이트 스스로 막아야 하는 구멍.**

**요구 수정(국소)**: 같은 call/dict 안에서 같은 필드가 2회 이상이면 ① 오류 기록((a) `duplicate`, (b) `invalid_module_gov` 또는 `duplicate`) ② `level` 은 **유효값 중 가장 엄한 것(strongest-wins)** 채택 — frozen>protected>watched, 유효값 없으면 기존대로 protected 보수. `reason`/`owner` 중복은 결정적이면 어느 쪽이든 무방(오류 기록은 필수).
**픽스처**: (a)(b) 각 1건 — 기대: level frozen(강한 쪽)+오류 기록. **음성검증**: strongest-wins 를 last-wins 로 되돌리면 스위트 FAIL 하도록 기대값이 물려 있을 것(재리뷰 시 직접 변조로 재현한다).

## 🔴 R-2 (보정 사유): `__gov__` 비정식 형태가 소리 없이 사라짐 (silent drop) — **계약 개정 포함**

**(a) 톱레벨 `AnnAssign`** — `__gov__: dict = {...}` / `__gov__: Final = {...}`
- 게이트 실측: `module: null`, 오류 0건 — **완전 무시.** `module_gov` 가 `ast.Assign` 만 본다.
- 실행 의미는 Assign 과 동일(값 있는 AnnAssign 은 런타임에 할당됨). 타입힌트 쓰는 코드베이스에서 지극히 자연스러운 표기라, **작성자가 frozen 을 선언했는데 게이트엔 무주석** — TASK-009 pass 로 흐른다. base∪head max 가드로도 못 잡는다(양측 모두 비가시).
- **계약 갭 인정**: A-0003 Q1(b)가 "톱레벨 직속 `Assign`"만 명시한 내 설계 누락이다(Codex 는 계약대로 구현). **이 문서로 계약 개정: 값이 있는 톱레벨 `AnnAssign`(단일 타겟 `__gov__`)도 정식 형태로 인정** — 파싱·검증 규칙은 Assign 과 동일.

**(b) 그 외 모든 `__gov__` 바인딩** — `if` 안, 클래스/함수 본문, `AugAssign`, 다중 타겟(`__gov__ = X = {...}`) 등
- 게이트 실측(`if True: __gov__ = {...frozen...}`): 완전 무시, 오류 0건.
- 선언 의도가 있는데 게이트가 못 본다는 사실 자체를 **기록 없이** 삼키면 silent pass 다. 하향 공격 경로는 아니지만(기존 톱레벨 선언은 계속 파싱됨), "민감 변경 포착" 목적에서 선언 유실은 동류의 구멍.
- **계약 개정: 정식 형태(톱레벨 Assign/AnnAssign 리터럴 dict) 밖의 `__gov__` 바인딩이 파일 어디서든 발견되면 `invalid_module_gov` 기록 + 모듈 protected 보수 취급.** false positive 는 보수 방향(최대 approval_required)이라 수용한다 — 조용한 유실보다 낫다.

**요구 수정(국소)**: `module_gov` 에 ① 톱레벨 AnnAssign 정식 처리 ② 트리 전체에서 그 외 `__gov__` 바인딩 탐지 시 `invalid_module_gov`+protected.
**픽스처**: (a) AnnAssign 정상 인정 1건 (b) 중첩 `__gov__` → `invalid_module_gov`+protected 1건 + 각각 음성검증 가능하게 기대값 고정.
