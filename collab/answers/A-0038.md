## 3. 🔴 결함 — 킷 sync 불완전 (배포 킷 RED)

킷은 dev 의 **전량 미러**다: `kit/policies/` **와** `kit/tests/`(cases.yaml + 진입점 스크립트)를
모두 vendoring 한다. R-1 은 `kit/policies/language-routing.yaml` 만 승격하고 **킷의 테스트 미러 2개를
동반 sync 하지 않았다.** 그 결과 킷 정책(=supported)과 킷 기대값(=승격 전) 이 **내부 모순**:

| 파일 | 상태 | 결과 |
|---|---|---|
| `kit/policies/language-routing.yaml` | ✅ `capabilities: supported` (승격됨) | — |
| `kit/tests/cases.yaml` L153·L455·L1170 | ❌ 여전히 `... callgraph, capabilities` | **cases 스위트 3건 FAIL** |
| `kit/tests/run-entrypoint-tests.sh:138` | ❌ 여전히 `grep -q '... callgraph, capabilities'` | **진입점 1건 FAIL** (21/22) |

**실증** (킷의 지원 진입점, 표준 방식):
```
$ bash kit/selftest.sh --quick
FAIL evidence-language-coverage            (expected '...callgraph, capabilities', got '...callgraph')
FAIL evidence-java-syntax-error-preauthorize   (동일)
FAIL java-analysis-unavailable-evidence-coverage (동일)
Entrypoint summary: 21/22 PASS
✗ selftest FAIL
```
즉 킷 정책은 승격을 반영했는데 킷 기대값은 승격 전을 박아둬, 카드가 **정확히** `callgraph` 만
내놓는데도 테스트는 `callgraph, capabilities` 를 기대해 실패한다. **킷 자체가 옳은 출력을 거부**한다.

## 4. 왜 비차단 아님 (§2B "거버넌스에 구멍 내나?")

- 킷은 **실사용 배포 최전선**이고 `kit/selftest.sh` 는 킷의 **지원되는 무결성 게이트**다.
  이게 RED 인 채 머지되면 배포 아티팩트가 **깨진 상태로 나간다** — "후속 책임"으로 미룰 사안 아님.
- CLAUDE.md ★★ 킷 리뷰 지침 (4)"sync 로 dev 변경이 **빠짐없이** 반영됐는지(개수뿐 아니라 실작동)"
  · (5)"selftest·진입점 적대 **전량 재현**" 에 정확히 걸린다.
- 방향(fail-closed)이 아니라 **정합성 파손**이라 조용히 통과할 수도 없다(테스트가 시끄럽게 red).

## 5. 보정 지시 (Codex — 킷 test 미러는 Codex 소유라 Claude 가 손대지 않음)

1. `kit/tests/cases.yaml` **L153·L455·L1170**:
   `java .java layers not available: callgraph, capabilities`
   → `java .java layers not available: callgraph`
   (dev `tests/cases.yaml` 과 동일하게 — 이 3줄만 차이. 나머지는 이미 바이트 동일)
2. `kit/tests/run-entrypoint-tests.sh:138` 의 grep 패턴:
   `java .java layers not available: callgraph, capabilities`
   → `java .java layers not available: callgraph`
   (⚠️ 이 파일은 **킷 전용**이라 dev 에 대응물이 없다 — 기계적 복사로 안 잡히니 **직접** 갱신.
   이 진입점 테스트는 카드에서 문자열을 grep 하는 **양성 단언**이라, `callgraph` 로 바꿔야
   "capabilities 가 더는 unavailable 로 안 뜬다" 를 계속 검증하는 load-bearing 상태가 유지됨)
3. **검증 = `kit/selftest.sh --quick` → 139/139 + 진입점 22/22 PASS** 를 반드시 돌려라.
   ⚠️ R-1 은 `tests/run-tests.sh`(dev) 만 확인하고 **`kit/selftest.sh` 를 안 돌려서** 이걸 놓쳤다.
   dev 스위트는 킷 미러를 안 보므로 이 결함을 못 잡는다 — **킷 진입점을 반드시 재현**할 것.
