## R-1 해소 확인 (A-0013 요청 보정 ①② 모두 채택)

**요청①(제어흐름 분리)** — 채택 ✓. `extract-gov-annotations.py` `merge_gov_annotations()`:

```python
        annotation_is_stronger = (
            strongest_level([merged["level"], annotation["level"]]) == annotation["level"]
        )
        if annotation.get("sink"):          # ← top-level if 로 분리 (강도병합과 독립)
            merged["sink"] = True
        if annotation_is_stronger:
            merged["level"] = annotation["level"]
            if annotation["reason"]: merged["reason"] = annotation["reason"]
            if annotation["owner"]:  merged["owner"] = annotation["owner"]
        else:                               # ← else 가 annotation_is_stronger 로 원복 (backfill 정상)
            if annotation["reason"] and not merged["reason"]: merged["reason"] = annotation["reason"]
            if annotation["owner"] and not merged["owner"]:   merged["owner"] = annotation["owner"]
```

sink 누적이 강도병합 `if/else` **밖 top-level if** 로 빠져 매 iteration 독립 실행 → `else` backfill 이 `annotation_is_stronger` 에만 결합(A-0013 요청 정확히 이행). sink 멤버십 정확성은 유지(top-level if), 약한 `sink=true` annotation 의 reason/owner backfill 복원.

**요청②(상설 회귀 픽스처)** — 채택 ✓. `tests/fixtures/gov-annotations/multi_sink.py` 신설:
```python
@gov(level="frozen")
@gov(level="protected", reason="PII bulk export", owner="security-reviewer", sink=true)
def download_bulk(): ...
```
케이스 `gov-annotations-multi-sink-metadata`(신규 harness 단언 `annotation_metadata`)가 병합 후 `sink:true` **AND** `reason:PII bulk export` **AND** `owner:security-reviewer` 동시 단언. §2B "고정 적대세트(데코레이터/오버로드)" 상설화. `sink=true`(Name 노드) 사용으로 하위호환 literal 커버리지도 확장.
