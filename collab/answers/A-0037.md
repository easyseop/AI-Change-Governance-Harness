## 9. 비차단 관찰 → **차기 AC 가드로 명시** (표류 금지)

CLAUDE.md §2B 절차대로 각 항목에 **"거버넌스 목적에 직접 구멍을 내나?"** 를 먼저 물었다.

### O-1 — 두 카탈로그를 **무조건 선(先)로드** ⇒ Java 없는 Python PR 이 승인요구로 뒤집힘
`check_new_capabilities()` 는 변경 경로에 `.java` 가 **하나도 없어도**
`metadata_by_language` 에서 Java 카탈로그를 **항상** 로드한다. 파일이 없으면 `FileNotFoundError` →
광역 `except Exception` → `approval_required`.

**실증(fresh repo, 무해한 2줄 Python PR)**:
| 조건 | 결과 |
|---|---|
| 두 정책 모두 존재 | `PASS` / exit 0 ✅ |
| `--java-policy` 부재 경로 | **`approval_required` / exit 2** ❌ |
| `--java-policy` 미지정(기본 상대경로 부재) | **`approval_required` / exit 2** ❌ |

**구멍인가?** 아니다 — **fail-closed(과탐) 방향**이라 민감 변경을 놓치지 않는다. 그리고 킷 배포
경로는 preflight 가 **먼저** 막아(§6, exit 2 + 명확한 메시지) 실제 사용자는 이 경로에 닿지 않는다.
⇒ **비차단**.
**그러나** 이것은 D-079 가 지목한 **"Java 무관 Python PR 전량 승인요구 = 게이트 신뢰 붕괴"**
실패모드가 *다른 문(파일 부재)* 으로 재진입한 것이다. AC#9 회귀 픽스처는 "두 정책 모두 존재"만
검증해 이 경로를 못 잡는다. 직접 CLI 호출자·레거시 자동화는 여전히 영향.
**해법은 3줄**(변경 경로에 해당 언어가 있을 때만 지연 로드).
⇒ **차기 태스크 AC 가드**: 언어 카탈로그는 **해당 언어 경로가 실제 변경됐을 때만 로드**하고,
**Java 카탈로그 부재 + 순수 Python PR → `pass`/exit 0 유지** 회귀 픽스처를 음성검증과 함께 추가.

### O-2 — `call_root_type` 이 **쓰레기 바인딩을 "해소됨"으로 취급** ⇒ 정직성 신호 억제
`unresolved_dynamic` 은 `not root_type` 일 때만 기록된다. 그런데 `variable_type_bindings` 는
Java 10+ `var` 선언에서 타입명을 문자 그대로 `"var"` 로 바인딩한다 —
**실증**: `var st = c.createStatement();` → `bindings == {'c':'Connection','u':'String','st':'var'}`.
`"var"`·`"Object"` 같은 **무정보 바인딩이 truthy** 라 (a) `calls` 매칭은 실패하고
(b) `unresolved_dynamic` 기록도 **억제**된다 ⇒ 미탐이 **조용해진다**.

**구멍인가?** 실측상 **아직 아니다** — 내가 miss 를 만들려 시도했으나 실패했다. 카탈로그가
**획득 지점(정적 팩토리)** 을 걸어두어(`Connection.createStatement`·`Cipher.getInstance`·
`Class.forName`·`HttpClient.newHttpClient`) 수신자가 `var` 여도 **획득 호출에서 잡힌다**.
위 `var` 케이스도 `sql_injection_surface` 정상 포착. ⇒ **비차단**.
**그러나** 이 안전성은 게이트 코드가 아니라 **카탈로그 작성 습관**에 의존한다(= 내가 정책
소유자로서 유지해야 하는 불문율). 획득 지점이 카탈로그에 없는 능력이 추가되는 순간
**조용한 미탐**이 되고, 설계가 금지한 "조용한 통과"가 된다.
⇒ **차기 태스크 AC 가드**: `var`·`Object` 등 **무정보 타입 바인딩은 미해소로 취급**하여
`unresolved_dynamic` 에 노출(정직성 우선). 픽스처로 고정 + 음성검증.

### O-3 — AC#8 의 "무시 사실을 coverage 에 남긴다" 부분 미이행 (경미)
AC#8 은 override 를 무시할 때 **그 사실을 coverage 에 노출**하라고 했으나, 현재는 조용히
기본 경로를 쓴다(`grep` 으로 coverage/override 키 0건). 게이팅 **동작 자체는 완전**(§5)하고
보안 목적(임의코드 실행 차단)은 달성됐으므로 **비차단**.
⇒ 차기 AC 가드에 **문구 이행**(무시 시 coverage 필드 1개) 포함.
