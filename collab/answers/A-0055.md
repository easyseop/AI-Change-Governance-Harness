## 3. 🔧 보정 방향 — **리뷰어가 워크트리에 직접 패치해 탐지축·소음축을 동시 실측했다**

두 갈래는 서로 못 고친다. **(a)·(b) 둘 다 필요**(각각 단독 측정 확인).

### (a) 타입 근거가 **아예 없는** deferred 레코드는 보수 승격 — R-1 의 **거울상**
R-1 은 *불투명 호출* 쪽에서 "전칭 → 존재" 로 고쳤다. 같은 오류가 **deferred 레코드 쪽**에 남아 있다: 근거가 **0** 인데 "불일치" 로 읽는다. **미상은 안전 방향으로 해석**해야 한다.

```python
def record_has_type_evidence(item, unresolved_by_caller):
    if item.get("dispatch_targets"):
        return True
    if item.get("receiver_type"):
        return True
    if item.get("kind") == "anonymous_class" and item.get("name", "").startswith("new "):
        return True
    return any(u.get("receiver_type") for u in unresolved_by_caller.get(item.get("caller"), []))
```
승격 조건에 `or not record_has_type_evidence(item, unresolved_by_caller)` 를 **추가**(기존 3분기 무개변).
- **`unresolved_by_caller` 갈래가 N4 를 지킨다**: N4 의 람다 레코드도 `receiver_type` 이 없지만 같은 caller `Flow.wire` 에 `consume`(`receiver_type: Flow`) 이 있어 **근거 있음** → 기존 `type_match` 판단으로 흘러 `pass` 유지. Q20 의 `Flow.wire` 는 unresolved 가 **하나도 없다** → 근거 0 → 승격. **정확히 갈린다.**
- **다른 구현을 택해도 좋으나 §3 측정표를 그대로 재현할 것.**

### (b) deferred 타입을 **리포 타입 계층 폐포**로 확장 — 상속 갈래
추출기는 이미 `declaration_supertypes()` 로 `supers` 를 계산하지만 **JSON 에 안 싣는다**. 실어서 게이트가 폐포를 뜨면 된다.
- 추출기 출력에 `"type_supers": {simple: [simple_super, …]}` 추가(`all_types` 의 `supers` 를 `simple_name()` 으로 정규화).
- 게이트에서 `deferred_types` 를 **상위타입까지 전이 폐포**로 확장한 뒤 교집합.
- 이 방향이면 Q11/Q11b 가 복구되고 **N4·Q9 는 `pass` 유지**(상위타입이 `String` 을 만들지 않는다).
