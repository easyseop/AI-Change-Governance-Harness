## 🟠 R-1 (블로킹) — `merge_gov_annotations` else 오결합으로 sink `owner`·`reason` 손실

**위치**: `.harness/gates/extract-gov-annotations.py` `merge_gov_annotations()` (헤드 기준 164~170행).

**무엇이 잘못됐나**: sink 처리 블록이 기존 `if annotation_is_stronger: … else: …` **사이에 끼어들어**, `else` 가 원래의 `annotation_is_stronger` 가 아니라 **새로 삽입한 `if annotation.get("sink")` 에 결합**됐다.

```python
        if annotation_is_stronger:
            merged["level"] = annotation["level"]
            if annotation["reason"]: merged["reason"] = annotation["reason"]
            if annotation["owner"]:  merged["owner"] = annotation["owner"]
        if annotation.get("sink"):          # ← 신규 삽입 (top-level if)
            merged["sink"] = True
        else:                               # ← 이 else 가 이제 sink 여부에 결합 (버그)
            if annotation["reason"] and not merged["reason"]: merged["reason"] = annotation["reason"]
            if annotation["owner"] and not merged["owner"]:   merged["owner"] = annotation["owner"]
```

원래 `else` 의미 = "이 annotation 이 더 약하면, 비어있는 reason/owner 를 backfill". 바뀐 의미 = "이 annotation 이 sink 가 **아니면** backfill" → **sink=True 인 (약한) annotation 의 reason/owner 는 backfill 이 스킵**된다.

**적대 실증(fresh · 픽스처 밖 · 다중 @gov 데코레이터)**:
```python
@gov(level="frozen")
@gov(level="protected", reason="PII export", owner="security-reviewer", sink=true)
def download(): ...
```
헤드 게이트 출력 → `reason=None owner=None sink=True` (**owner·reason 유실**).

**음성검증(rig-and-revert)**: sink 라인을 if/else 밖으로 빼 `else` 를 `annotation_is_stronger` 에 원복하면 → `reason="PII export" owner="security-reviewer" sink=True` (정상 복원). = 오결합이 load-bearing 결함임을 실증(우연 아님).

**왜 비차단으로 흘리지 않나(§2B 필수질문)**: sink **집합 멤버십**은 항상 정확(`merged["sink"]` 는 매 iteration top-level if 로 설정 → sink 누락/오탐 없음). 그러나 손실되는 `owner` 는 **§3.1·§5 가 sink 라우팅 대상으로 명시한 필드**(TASK-024 라우팅 입력), `reason` 은 감사카드 30초 판독 필드다. 다중 @gov(또는 sink 옵트인이 약한 레벨) 케이스에서 라우팅 owner 가 조용히 None 으로 떨어지면 후속 라우팅이 fail-safe(tool_owner)로 강등된다. **거버넌스 메타데이터 손실 + 명백한 제어흐름 버그**라 "논리 정합" 바를 못 넘는다(수용기준 충족만으로 통과 불가 — `COMMON-RULES` §1·CLAUDE §2B). 또한 이 결함은 CLAUDE §2B 가 **상설 적대세트로 매번 돌리라 명시한 "데코레이터/동명 오버로드"** 범주에 정확히 위치하는데 회귀 픽스처가 없다.

**요청 보정**:
1. sink 설정을 `if annotation_is_stronger: … else: …` **밖으로 분리** — 예: 루프 상단에 `if annotation.get("sink"): merged["sink"] = True` 를 두고, 기존 stronger/else backfill 은 원래대로 복원. (backfill 은 `annotation_is_stronger` 에만 결합.)
2. **상설 회귀 픽스처 추가**(§2B "고정 적대세트"): 함수 하나에 **다중 @gov 데코레이터**(약한 쪽이 `sink=true`+reason/owner, 강한 쪽은 무-reason) → 병합 후 `sink=True` **AND** reason/owner 보존을 단언하는 케이스. 이 픽스처가 오결합 회귀를 잡도록 음성검증 가능해야 한다.
