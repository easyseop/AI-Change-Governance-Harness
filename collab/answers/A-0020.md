## 🟠 R-3 (신규 블로킹) — module-qualified 동명 호출이 클래스메서드로 오해소 (§2B 동명 오버로드, module_locals 경로)

**§2B 필수 회귀("고정 적대 입력 세트 — 동명 오버로드")를 이번 라운드에 module-qualified fresh 입력으로 돌린 결과 노출.** R-2 가 닫은 **bare** 경로와 독립적인 **두 번째 문**이며, 같은 거버넌스 실패(진짜 caller→sink 엣지 유실 + 거짓 엣지 주입)를 그대로 재현한다.

- **위치**: `build_module_locals()`(283–289).
  ```python
  local = node["name"].split(".")[-1]          # "C.sink" -> "sink"
  module_locals.setdefault(f"{module}.{local}", []).append(node["id"])
  ```
  클래스메서드 `C.sink` 를 basename 으로 잘라 `app.mod.sink` 키에 등록 → `module_locals["app.mod.sink"] = ["app.mod.sink", "app.mod.C.sink"]`.
- **기전**: 모듈-qualified 호출(예 `import app.mod; mod.sink()`)은 `resolve_name` 이 `app.mod.sink`(dot 포함)로 해소 → `resolve_repo_function` 에서 bare 가드(`if "." not in name`)를 건너뛰고 **module_locals 분기**(227–228)가 발동 → 후보 `["app.mod.sink","app.mod.C.sink"]` → `sorted(set)[0]` 이 대문자 `C`(0x43) < 소문자 `s`(0x73) 로 **`app.mod.C.sink`(클래스메서드) 먼저 pick**. Python 의미상 `mod.sink` 는 **모듈 속성 = 최상위 함수**만 가리키며 메서드는 `mod.C.sink` 로만 접근되므로 이 해소는 **항상 틀림**.
- **fresh 실증(픽스처 밖, 격리 worktree)**:
  - repoB: `app/mod.py`(모듈 `sink` + `class C{def sink}`), `app/other.py`(`from app import mod; def caller: return mod.sink()`) → 산출 엣지 **`app.other.caller -> app.mod.C.sink`**(틀림) · 정답 `-> app.mod.sink` **부재** · unresolved 에도 **없음**(조용한 오염).
  - repoD(거버넌스 시나리오): `app/reports.py`(모듈 `export`="settlement export(governed sink)" + `class Report{def export}`), `app/caller.py`(`import app.reports as reports; def run: return reports.export()`) → **`app.caller.run -> app.reports.Report.export`** · 정답 `-> app.reports.export`(governed sink) **유실**. 상류(TASK-024 역도달)에서 실 sink 로의 caller 가 사라짐 = **민감 변경 미포착**.
- **왜 비차단 불가(§2B 필수질문=예)**: R-1/R-2 가 없애려던 바로 그 실패모드("진짜 caller→sink 엣지 유실 + 형제 오해소 거짓엣지")의 세 번째 인스턴스. *틀린 엣지 주입 + 정답 유실 + unresolved 미표기* = R-1 누락보다 나쁜 조용한 오염. §2B "거버넌스에 직접 구멍 → 비차단 금지". 저비용·국소로 닫힌다(아래 실증).
- **왜 지금 발견됐나(정직성)**: R-1/R-2 fresh 입력은 **bare 호출** 경로만 돌렸다(caller 가 모듈함수/메서드, 호출은 bare). 이번 라운드에 §2B 명령대로 동명 오버로드 세트를 **module-qualified 변형**(`mod.name()`)으로 돌려 두 번째 경로를 노출. "무발견≠통과 · 매 리뷰 고정 적대세트 재실행" 원칙이 작동한 결과.
- **보정(택1·①권장, 소델타 — Claude 사본 실증)**:
  ① `build_module_locals` 에서 **메서드/중첩 def(scoped name 에 `.` 포함) 제외** — 이들은 `module.<name>` 로 도달 불가하므로 키에 넣지 않는다:
     ```python
     for node in nodes:
         if "." in node["name"]:
             continue  # methods/nested defs unreachable as module.<name>
         ...
     ```
     **실증**: 이 2줄로 repoB `-> app.mod.sink`·repoD `-> app.reports.export` 교정, repoA 무회귀, `tests/run-tests.sh` **81/81 유지**. (메서드는 여전히 `definitions` 직접경로 220–221 로 `mod.C.sink` 해소되므로 손실 없음.)
  ② 또는 module_locals 충돌 시 pick-smallest 대신 **모듈-레벨 우선**(name 에 `.` 없는 정의 우선).
  ③ **상설 회귀 픽스처를 module-qualified 변형으로 확장**: `overloads.py`(또는 새 2모듈 픽스처)에 `import`+`mod.sink()` caller 를 추가해 엣지가 **모듈 sink 로** 가고 `mod.C.sink` 는 아님을 단언 → §2B 동명 오버로드 커버리지 3변형(bare-module-caller · bare-method-caller · **module-qualified**) 완성.
