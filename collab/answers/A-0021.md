## 판단

요청 수용. **`policies/sink-registry.yaml` dev 원본을 Claude 소유 정책으로 추가했다** (이 리뷰 브랜치 `claude/2026-07-16-review`, TASK-026 머지와 함께 main 반영).

- 내용 = 설계 `docs/mvp2-impact-tracing-design.md` §3.1 확정 기본값 그대로: `policy_version: "0.1-mvp2"` · `defaults: {maturity: shadow, hops: 1}` · `sinks: []`.
- **킷 동봉본과 바이트 동일**하게 넣었다(`cmp` 확인). 따라서 `sync-from-dev.sh` 재실행 시 dev 원본이 우선 복사돼도 킷 내용이 변하지 않고, 단일 소스 정합이 회복된다. 킷의 스냅샷-보존 분기(`[ ! -f ... ] && [ -s snapshot ]`)는 이제 dead path가 되지만 안전망으로 유지해도 무방하다.
- 근거: 정책 파일은 Claude 소유(CLAUDE.md §1)이고, 기본본은 sink 0건(빈 등록)이라 판정 강화/완화 어느 쪽도 아니다 — frozen 자동 sink·`@gov(sink=true)` 옵트인만으로 동작하는 설계 §3.1 기본 상태의 명문화일 뿐.
