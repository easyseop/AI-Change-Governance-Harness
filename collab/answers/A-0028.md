## 2. R-5 (보정요청) — **새 필수 정책 preflight 가 죽은 가드 + 레거시 override 무예고 파괴**

### 2.1 관찰 — RIG-2 음성검증에서 가드가 죽어 있다

`kit/run.sh` preflight 목록에서 `"$LANGUAGE_ROUTING"` 만 제거(다른 변경 없음):

```
for required_policy in "$ZONES" "$CAPS" "$ROUTING" "$SINKS"; do   # LANGUAGE_ROUTING 제거
→ 킷 진입점 14/14 PASS  (전부 초록)
```

**이 새 가드를 통째로 지워도 어떤 시험도 울지 않는다.** RIG-1 은 13/14 로 울었는데 RIG-2 는 무음 —
즉 보정 커밋이 추가한 두 가드 중 **하나만 하중을 받고 하나는 회귀 방지가 없다.**

### 2.2 이 가드가 실제로 막고 있는 것 (= 지워지면 무엇이 돌아오나)

무음이라고 무의미한 게 아니다. **가드를 제거한 상태에서** 레거시 override 디렉터리
(구 킷 기준 정책 4종만 있고 `language-routing.yaml` 은 없는, 업그레이드 전에는 완전 정상이던 배포)로
무해한 Python 1줄 변경을 돌리면:

```
$ kit/run.sh HEAD~1..HEAD --repo <legacy> --policies <legacy>/policies
exit=1
  verdict: blocked
  reasons:
    - 'language routing policy missing: <legacy>/policies/language-routing.yaml'
```

**R-4 와 동일 계열의 오판정**(무해 변경 → 🔴 BLOCKED)이 위치만 "대상 repo cwd" 에서
"레거시 override 디렉터리" 로 옮겨 그대로 재현된다. preflight 가 있을 때만 이게
`✗ 분석 실패: 필수 정책 파일 없음: …` 라는 **정직한 도구 실패**로 강등된다.

즉 이 한 줄은 **배포 최전선에서 R-4 재발을 막는 유일한 방벽인데, 회귀 시험이 0 이다.**
A-0026 R-1 에서 "유령항목 가드가 죽어 있어 rig 해도 무음" 을 보정요청한 것과 **정확히 같은 패턴**이며,
CLAUDE.md §2B "거버넌스에 직접 구멍 내는 관찰은 비차단으로 흘리지 말 것" 에 해당한다.

### 2.3 부수 관찰 — 업그레이드 마이그레이션 안내 부재

`kit/README.md` 는 override 사용법을 이렇게만 적는다:

```
./run.sh <base>..<head> --repo <대상repo> --policies <대상repo>/policies
```

**필수 파일 집합이 어디에도 없다.** `language-routing.yaml` 은 `kit/manifest.yaml:66` 에만 등장한다.
그 결과 기존 도입처가 킷을 올리는 순간 **모든 실행이 분석 실패**로 죽는데, README 만 보고는
"내 정책 디렉터리에 파일 하나를 추가해야 한다" 를 알 길이 없다. fail-closed 자체는 옳은 선택이지만
(무음 폴백은 coverage 를 조용히 잃으므로 더 나쁘다), **예고 없는 fail-closed 는 배포 사고다.**
