## R-1 (차단) — 미선언이 **1층 frozen 차단을 무력화**한다 (under-detection)

### 무엇이 문제인가

`build_evidence()` **첫 줄**이 `load_intent()` 다:

```python
def build_evidence(args):
    intent = load_intent(args.change_intent)      # ← 여기서 IntentNotDeclaredError
    sensitive_policy = load_sensitive_policy(...)  # ← 도달 못 함
    ...
    sensitive_check = sensitive_result(files, sensitive_policy)   # 민감경로층 미실행
    function_gov = build_function_gov_result(...)                 # @gov 함수층 미실행
```

미선언이면 예외가 `main()` 에서 잡혀 `intent_not_declared_evidence()` 로 **카드 전체를 대체**한다. 결과: **의도층만 건너뛰는 게 아니라 민감경로층·@gov 함수층까지 통째로 미실행**된다.

그런데 `run.sh` 에서 **1층 판정(의도·민감경로·@gov)은 오직 `generate-change-evidence` 하나가 낸다**(`ge_exit`). 별도 `check-sensitive-zones` 실행이 없다. 따라서 카드 게이트가 abort 하면 **frozen 차단을 낼 주체가 사라진다.**

### 실증 — `change-intent.yaml` 삭제만으로 🔴BLOCKED → 🟠APPROVAL

fresh 적대 repo(픽스처 밖·`app/settlement/core.py` = **frozen 정산·금전 직접영향** 수정, intent 미선언):

| | main (before) | **브랜치 (제출본)** |
|---|---|---|
| `kit/run.sh` 최종 | 🔴 **BLOCKED (exit 1)** | 🟠 **APPROVAL_REQUIRED (exit 2)** |
| 카드 `verdict` | blocked | approval_required |
| `frozen_touched` | `[]` | `[]` |
| `changed_files` | `[]` | `[]` |

**두 번째 벡터도 동일** — `@gov(level="frozen", reason="이자 계산 핵심")` 함수 본문 변경 + 미선언:

| | main | **브랜치** | (보정 프로토타입) |
|---|---|---|---|
| exit | **1** | **2** | **1** |
| `changed_functions` | `[]` | `[]` | `function_frozen:app/biz.py::compute_interest:base:…` |

⇒ **정산·이자(🔴 금전 직접영향)를 고치면서 `change-intent.yaml` 만 지우면 차단이 승인요구로 내려간다.** 킷 게이트도 md5 동일이라 **배포 최전선에 그대로 실린다**(kit gate 실측 exit 2).

### 왜 비차단으로 못 미루나 (§2B 필수질문)

> "이게 거버넌스 목적(민감 변경 포착)에 직접 구멍을 내나?" → **예, 결정적으로.**

- 하네스 **최상위 불변식 = "1층 frozen 만 차단"**. 그 유일한 차단을 미선언이 무력화한다.
- 더 나쁜 건 **사람이 알 수 없다는 것**: 카드에 `frozen_touched: []`·`changed_files: []`·`files_changed: 0` 뿐이라, 승인자는 **정산 코드가 바뀐 사실 자체를 볼 수 없다**. "intent 선언 안 했네, 승인" 하면 금전 로직이 그대로 통과한다. 카드는 CI 감사 산출물이고 킷은 배포 최전선이다.
- 방향이 **과소탐(under-detection)** 이다. 과탐이면 감내하지만 과소탐은 안 된다.

### AC 해석 — 어디서 어긋났나

AC#1 의 "`blocked` 금지(1층 frozen 아님)" 는 **미선언 *자체* 를 frozen 취급하지 말라**는 뜻이다(= 무해 변경 + 미선언 → 2). **"frozen 을 건드렸는데 미선언이면 frozen 검사를 하지 말라"** 는 뜻이 결코 아니다. AC#2 도 **의도층** 미검사만 요구했지 `sensitive_zone_check: not_checked` 를 요구하지 않았다. 민감경로층·@gov 층은 **intent 를 입력으로 쓰지 않는다** — 건너뛸 이유가 없다.

## R-2 (차단·R-1 과 동일 뿌리) — 카드가 **사실과 다르다** (AC#2 정직성 위반)

미선언 카드는 `summary.files_changed: 0` · `changed_files: []` 라고 **주장**한다. 실제로는 파일이 바뀌었다(fresh repo `git diff --name-only` = **1**). D-065 가 잡은 "합성 선언으로 pass 위조" 는 사라졌지만, **"변경 없음"이라는 다른 형태의 허위**가 들어왔다. `base_commit: unknown`·`policy_sha: {}`·`generated_on: '1970-01-01'` 도 마찬가지로 계산 가능한데 버려졌다.

**픽스처가 이 허위를 계약으로 고정한 게 더 심각하다**: `missing-change-intent` 케이스 입력은 `tests/fixtures/good/name-status.txt`(= `M app/features/search.py`, **1개 변경**)인데 기대값이 `files_changed: 0`·`changed_files: []` 다. 즉 **테스트가 "카드는 눈감아도 된다"를 못박았다.** 그래서 96/96 이 초록인데도 구멍이 남는다 — 무발견이 아니라 **미측정**이다.

정직성의 정의: **"검사 안 한 것을 검사했다고 하지 않는다"** + **"본 사실을 안 봤다고 하지 않는다"**. 후자가 깨졌다.

## 보정안 (실증 완료 — 프로토타입으로 두 AC 동시 충족 확인)

**핵심 = abort 하지 말고, 의도층만 `not_declared` 로 표시하고 파이프라인을 계속 돌린다.** (합성 선언 주입은 **절대 금지** — D-065 R-1 재발. 아래는 `intent_result` 를 빈 선언으로 돌린 뒤 **status 를 not_declared 로 덮어써** pass 주장을 차단한다.)

1. **`build_evidence` 에서 미선언을 흡수**:
   ```python
   intent_declared = True
   try:
       intent = load_intent(args.change_intent)
   except IntentNotDeclaredError:
       intent_declared = False
       intent = {"requirement_id": None, "author": None,
                 "allowed_paths": [], "forbidden_paths": [], "expected_paths": []}
   ```
2. **의도층 판정을 not_declared 로 고정**(빈 선언이 `out_of_scope` 과탐을 내지 않도록 비우고, `status` 로 미검사 사실을 노출):
   ```python
   if not intent_declared:
       intent_check = {**intent_check, "status": "not_declared",
                       "out_of_scope_paths": [], "forbidden_touched": [],
                       "expected_paths": [], "missing_expected": []}
   ```
3. **verdict 우선순위**(frozen 우선·미선언은 approval 하한):
   ```python
   if intent_check["forbidden_touched"] or sensitive_check["frozen_touched"]:
       return "blocked", BLOCKED
   if intent_check.get("status") == "not_declared":
       return "approval_required", APPROVAL_REQUIRED
   ```
4. `intent_not_declared_evidence()` **삭제**(카드 대체 경로 자체를 없앤다). `IntentNotDeclaredError` 는 유지 가능.
5. **AC#3 유지**: 선언이 없으면 `in_allowed_paths` 는 `true` 가 될 수 없다. 프로토타입은 `false` 가 나왔고 AC 허용범위지만 **`null` 을 권장**한다 — "선언 밖"이 아니라 "대조할 선언이 없음"이 사실이다(템플릿 명기).
6. **coverage 정직성 유지**: `checked` 에 `check-change-intent` 를 넣지 않는다 + `not_checked` 에 `intent_not_declared` 토큰 유지(현행 문구 그대로 좋다). 단 **민감경로·@gov 는 실제로 검사했으므로 `checked` 에 정상 등재**돼야 한다(지금은 통째로 `checked: []` → 이건 이제 거짓이 된다).
