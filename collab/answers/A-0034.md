## 🔴 R-1 (보정필수) — Java **파싱 실패 파일이 gov_level 축을 조용히 통과**시킨다 (AC#4 위반 · Java↔Python parity 파손)

### 증상 — 배포 킷 end-to-end 실증
동일한 `@PreAuthorize("hasRole('ADMIN')") → ("hasRole('USER')")` **인가 약화**를, 유효한 `change-intent.yaml` 을 선언한 fresh repo 에서 `kit/run.sh` 로 통째 돌린 A/B:

| | 같은 파일에 파싱 불가 메서드 1개 추가 | 판정 | 카드 `reasons` |
|---|---|---|---|
| **A** | 있음 (`public void oops( { }`) | **🟢 PASS · exit 0** | **`[]` (비어 있음)** |
| **B** | 없음 | 🟠 APPROVAL_REQUIRED · exit 2 | `function_protected:app/S.java::S.wire:base` + `:head` |

**즉, 망가진 메서드 한 개를 덧붙이면 인가 경계 변경이 승인요구 → 무사통과로 바뀐다.** 카드에 coverage 한 줄(`java gov_level analysis unavailable for app/S.java: syntax error`)은 남지만, **하네스가 산출하는 판정 자체가 틀린다** — 그리고 사람은 판정을 보고 움직인다.

### 원인 (한 줄 단위)
`check-function-gov-level.py`

1. `java_result_from_source()` 가 `except (SyntaxError, ImportError, OSError)` **세 가지를 한 메시지**(`"java analysis unavailable: {error}"`)로 뭉갠다.
2. `java_analysis_unavailable()` 이 그 문자열을 **부분일치**로 판정한다.
3. 본 루프에서 참이면 `coverage_not_checked.append(...)` 후 **`continue`** — 즉 `parse_failed()` 기반 **fail-closed 블록(`fail_closed_record(... level=base_level or "protected")`)에 도달하기 전에 파일을 건너뛴다.**

Python 형제 경로는 같은 조건에서 **`approval_required` exit 2** 를 낸다(fresh repo 로 실증). 같은 결함 종류에 두 언어가 **반대 판정**을 내는 parity 파손이다.

### 왜 이게 "AC 해석 차이"가 아니라 결함인가
AC#4(`parse_error → 최소 approval · 조용한 pass 금지`)와 A-0033/AC#9(e)(`파서 부재 → 미분석 노출 · exit 0`)는 **서로 다른 두 조건**을 말한 것인데 구현이 하나로 합쳤다. 정책 소유자로서 경계를 명시한다:

- **툴체인 부재**(`ImportError`/`OSError` — tree-sitter 미설치): 환경 조건이라 **모든** `.java` 에 균일하게 걸린다. 의존성 미설치를 이유로 전 PR 을 승인요구로 만드는 것은 D-074·D-076 이 의도적으로 피한 실패다 → **exit 0 + coverage 노출이 옳다. 현 구현 유지.**
- **파일별 구문오류**(`SyntaxError` — `root.has_error`): **내용 특정**이며 그 파일의 `@Gov`·`@PreAuthorize` 를 임의로 숨길 수 있다 → **AC#4 그대로 `approval_required`(exit 2). 절대 blocked 아님.**

전자는 "아무것도 못 봤고 그렇게 말했다"(정직), 후자는 "이 변경 파일이 인가 어노테이션을 숨겼는데 통과라고 말했다"(허위).

### 보정 지시 (최소 diff)
1. `java_result_from_source()` 에서 예외 종류를 **구분해 보존**한다 — 예: `parse_error` 는 유지하되 `unavailable_kind: "toolchain" | "syntax"` 를 함께 싣는다. **문자열 부분일치 판정 금지**(메시지 문구 바뀌면 조용히 무너진다).
2. 본 루프의 skip/`continue` 는 **`toolchain` 인 경우에만** 적용한다. `syntax` 는 `continue` 하지 말고 기존 Python 과 **동일한 fail-closed 경로**로 흘려보낸다(`base_level or "protected"` 승계 포함).
3. **회귀 픽스처 추가**(현재 이 경로 **0건 = 미측정**이라 결함이 통과했다):
   - (a) `.java` head 구문오류 + base 에 `@PreAuthorize` → **`approval_required` exit 2**, 카드 `reasons` 에 해당 함수 노출.
   - (b) 툴체인 부재(기존 `shims/tree_sitter.py` 재사용) → **exit 0** + coverage 노출 (현행 유지 회귀 고정).
   - (c) 각각 **음성검증**: 기대값 변조 시 **단독 FAIL**.
4. **parity 픽스처 확장**(AC#7): 현행 parity 는 `@Gov` 대칭 케이스만 단언해 이 파손을 못 잡았다. **"구문오류 파일 → 양 언어 동일 verdict"** 쌍을 `tests/parity/` 에 추가할 것.
