- **AC #2 동적성**: `checked` 목록이 **실제 실행 술어**(`can_run_function_gov`) 기반. baseline(name-status 파일 입력, function-gov 미실행)=2게이트 vs function 케이스(base..head ref, function-gov 실행)=3게이트로 정직하게 갈림. **음성검증 성립**: `executed_gate_records`를 `if True`로 변조해 function-gov를 상시 노출시키면 baseline 케이스 FAIL(`expected 2 gates, got 3`) — 동적성 가드가 실가드임을 확인함.
- **AC #3** `python_version`(`sys.version.split()[0]`) 추가 + `tool_version`·`policy_sha`(sha256 64hex) → `version_metadata_present` 테스트로 고정.
- **AC #4** 정상카드 top-level 키 == 템플릿 키(17개) → `schema_keys_match_template` 테스트. 이 태스크가 누락돼 있던 `changed_functions` 를 템플릿에 반영해 정상카드↔템플릿 정합을 맞춘 것은 타당.

## 🔴 보정 R-1 — **예외/오류 카드가 실행되지 않은 게이트를 "검사했다"고 위조** (AC #2 위반)

**무엇**: `main()` 의 예외 핸들러(=`build_evidence` 가 `load_intent` 등에서 throw한 오류 카드)가 `coverage_statement(args.diff_input, "blocked")` 를 그대로 호출한다. 이 경로에선 **게이트가 하나도 실행되지 않았는데** 카드는 다음을 출력한다:

```yaml
verdict: blocked
coverage_statement:
  verdict_statement: governance violation detected      # ← 실제론 '입력/설정 오류'이지 위반 탐지가 아님
  checked:
    - gate: check-change-intent      # ← 안 돌았다
    - gate: check-sensitive-zones    # ← 안 돌았다
    - gate: check-function-gov-level # ← 안 돌았다
policy_sha: {}                        # ← 이건 정직하게 비어있음 (아무것도 해시 안 함)
reasons: ['의도 선언 누락: change-intent.yaml 파일을 찾을 수 없습니다.']
```

**실증(fresh 입력, 픽스처 밖)**: `generate-change-evidence.py HEAD~1..HEAD --change-intent /nonexistent.yaml` → 위 카드 그대로 재현. 게이트 0개 실행인데 `checked` 는 3게이트를 나열.

**왜 결함(§2B)**: 이 태스크의 **거버넌스 기능 자체가 "카드 정직화"** 다. AC #2 는 명시적으로 *"정적 문구가 아니라 실행된 게이트 기반이어야 함 — 게이트를 빼고 돌리면 본 것이 줄어야 정직"*. 오류 카드는 **실행되지 않은 게이트를 검사했다고 단언** → 정확히 이 태스크가 없애려는 안티패턴이며, **이 태스크의 신규 coverage 블록이 도입한 정직성 회귀**다. 내부 모순이 의도를 증명한다: 같은 오류 카드가 `policy_sha:{}` 로 "아무것도 안 함"을 정직하게 비우면서 `checked` 만 3게이트를 위조.

**도달 가능성 높음**: 루트에 `change-intent.yaml` 없이 `--change-intent` 미지정 기본호출, 정책 YAML 파손, git 오류 등에서 상시 진입. (실제로 `tests/fixtures/evidence-baseline/change-intent.yaml` 은 존재하지도 않음 — 의도파일 부재는 흔한 상태.)

**수정계약(국소·저비용)**: 예외 경로에서 **실행된 게이트가 0개이면 `coverage_statement.checked` 는 `[]`** 여야 한다(주변의 `policy_sha:{}`·`changed_files:[]`·`summary:0` 처럼 정직하게 비움). 정상 경로 로직은 **건드리지 말 것**(`combine_verdicts`→ blocked인 진짜 frozen 케이스의 `governance violation detected` 는 정확하므로 유지). `verdict_statement` 를 오류 카드에서 중립 문구로 바꾸는 건 선택(권장하나 비필수) — **필수는 `checked:[]`**. 회귀 픽스처(오류 카드 → `checked:[]`) + 음성검증(기대 변조 시 FAIL) 추가.

## 🟠 보정 R-2 — **오류 카드 스키마 드리프트: `changed_functions` 누락** (AC #4 위반)

**무엇**: 예외 카드 dict 는 `changed_functions` 키가 **없다**(top-level 16키). 그러나 템플릿·정상카드는 `changed_functions` 포함(17키). 이 태스크가 템플릿에 `changed_functions` 를 추가하면서 오류 카드에는 반영하지 않아 **template↔오류카드 불일치**를 남겼다.

**실증**: 오류 카드 키 = 템플릿 키 − `{changed_functions}`.

**테스트 사각지대**: `schema_keys_match_template` 가드는 정상경로 픽스처 4건에서만 돈다 → 오류 카드를 **한 번도 검사하지 않아** "스키마 == 템플릿"이라는 **거짓 확신**을 준다.

**수정계약**: 오류 카드 dict 에 `changed_functions: []` 추가(정상카드·템플릿과 정합). 그리고 **오류경로 픽스처 1건**을 추가해 `schema_keys_match_template`(및 R-1 의 `checked:[]`)를 **오류 카드에도 실제 적용**해 가드가 오류 경로를 커버하게 할 것.
