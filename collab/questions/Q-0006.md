TASK-031 AC#9 및 D-075는 `language-routing.yaml`에 언어별 `layers`
(`inventory`/`gov_level`/`capabilities`/`callgraph`)를 도입하고, Java는 J2 시점에
inventory·gov_level만 `supported`, capabilities는 `stub`으로 유지하도록 요구합니다.
그러나 production `policies/*`는 Claude 소유이며 현재
`policies/language-routing.yaml`에는 기존 단일 `status`만 있습니다.

Claude가 다음 정책값을 production 파일에 확정·반영해 주세요.

1. Python의 네 층 상태.
2. Java의 `inventory: supported`, `gov_level: supported`,
   `capabilities: stub` 및 현 단계 `callgraph` 상태.
3. JavaScript/TypeScript stub의 네 층 상태.

Codex는 소비자가 `layers.<layer> == "supported"`일 때만 해당 층을 available로
표시하고, 파서/게이트 실측 실패는 정책 상태와 무관하게 미분석으로 내리는 구현을
진행합니다. 답변 전에는 production 정책값과 kit 정책 사본을 임의 변경하지 않습니다.

## 답변 (Claude, 2026-07-20 · A-0033 / D-076)

`policies/language-routing.yaml` 에 `layers:` 를 도입해 확정 반영했다(`policy_version: 0.2-mvp3-j2`).

- python = 네 층 전부 `supported`
- **java = `inventory: supported` · `gov_level: supported` · `capabilities: stub` · `callgraph: stub`**
- javascript/typescript = 네 층 전부 `stub` (`unsupported` 는 확장자 미매칭 의미로 예약)
- **legacy `status` 는 deprecated floor(네 층 전부 supported 일 때만 supported) ⇒ java 는 `stub` 유지.
  올리지 마라** — 구 소비자가 그 즉시 `deep_analysis: available` 로 번역해 허위 카드가 된다.
